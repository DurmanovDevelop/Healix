Healix AI — Информационная система анализа медицинских показателей

    Дипломный проект
    Разработка информационной системы мониторинга персональных медицинских показателей с применением методов машинного обучения и искусственного интеллекта.

🩺 О проекте
Healix AI — это полнофункциональное веб-приложение (Fullstack), которое позволяет пользователям отслеживать состояние здоровья, получать прогнозы рисков на основе ML-моделей и консультироваться с ИИ-ассистентом. 
Система автоматически распознает медицинские показатели из обычного текста в чате (например, "у меня давление 140 на 90 и пульс 80") и мгновенно обновляет дашборд пользователя, сохраняя данные в реляционную базу данных.
✨ Ключевые возможности

    Ролевая модель доступа (RBAC): Пациент, Врач, Администратор.
    Healix AI Ассистент: Уникальный чат для каждого аккаунта с историей переписки. Распознает симптомы и числовые показатели.
    ML-Анализ: Встроенные модели прогнозирования рисков (Сердечно-сосудистые заболевания, Диабет, Общий скоринг здоровья).
    Панель врача: Управление списком пациентов, добавление новых, просмотр диагнозов и уровней риска.
    Панель админа: Статистика системы, управление пользователями и аудит.
    Безопасность: JWT-аутентификация, хеширование паролей (bcrypt), проксирование запросов к ИИ через бэкенд (ключ API не виден на фронтенде).

📋 Требования перед запуском
Убедитесь, что на вашем компьютере установлены:

    Python 3.10 или выше (при установке обязательно поставьте галочку "Add Python to PATH").
    Node.js 18+ и npm (для работы фронтенда).
    PostgreSQL 16 (или запущенный Docker-контейнер с PostgreSQL).

🚀 Пошаговая инструкция по запуску
Шаг 1: Настройка базы данных (PostgreSQL)

    Откройте pgAdmin или командную строку psql.
    Создайте базу данных с именем medical_analysis_db.
    Пароль пользователя postgres должен быть установлен в 1342 (или измените его в файле backend/.env).
    Выполните SQL-скрипты из папки database/ по порядку:
        Сначала 01_schema.sql (создаст 15 таблиц).
        Затем 02_seed.sql (заполнит справочники и демо-данные).

Шаг 2: Настройка и запуск Backend (Python)
Откройте терминал (PowerShell или CMD) и выполните команды:

bash
# Переходим в папку бэкенда
cd backend

# Создаем виртуальное окружение
python -m venv venv

# Активируем его (для Windows)
venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt

# (Опционально) Создаем демо-пользователей врача и админа, если их нет в БД
python add_demo_users.py

# Запускаем сервер FastAPI
python run.py

✅ Сервер запущен на http://localhost:8000. Документация API доступна по адресу http://localhost:8000/api/docs
Шаг 3: Настройка и запуск Frontend (React)
Откройте второй, новый терминал (не закрывая первый) и выполните:

bash
# Переходим в папку фронтенда
cd frontend

# Устанавливаем npm-пакеты
npm install

# Запускаем сервер разработки
npm run dev

✅ Фронтенд запущен на http://localhost:5173

💡 Советы для защиты диплома
Если вас спросят про архитектуру или реализацию, вот ключевые моменты, которые стоит упомянуть:

    Нормализация БД: Схема базы данных приведена к третьей нормальной форме (3NF) для исключения аномалий обновления и избыточности данных.
    Безопасность API: Ключ доступа к нейросети (HEALIX_API_KEY) хранится только на сервере в файле .env. Фронтенд никогда не взаимодействует с внешним ИИ напрямую, что предотвращает утечку ключа.
    Интеллектуальный парсинг: Реализован гибридный подход: регулярные выражения на фронтенде резервируют извлечение показателей, а сам ИИ (через системный промпт) также возвращает структурированный JSON [INDICATORS_UPDATE: {...}] для гарантированной синхронизации с базой данных.
    Изоляция данных: История чатов привязывается к user_id и хранится локально с префиксом, гарантируя, что один пользователь не увидит переписку другого.

❓ Частые проблемы (Troubleshooting)

    Ошибка 401 Unauthorized при входе: Убедитесь, что вы выполнили скрипт add_demo_users.py или 02_seed.sql, и что в backend/.env правильный пароль БД (1342).
    Ошибка Port 8000 is already in use: Другая программа использует порт 8000. Закройте её или измените порт в backend/run.py (например, на 8001).
    Фронтенд не видит бэкенд: Проверьте, что в frontend/vite.config.js прописан прокси на http://localhost:8000, и что сервер Python запущен в отдельном окне терминала.
